STM32F411RET6 定时器中断控制LED闪烁工程
 
项目概述
 
本项目基于STM32F411RET6主控芯片，采用通用定时器TIM2中断机制，实现板载PA5引脚LED的精准周期性闪烁，替代传统阻塞式延时函数，完成嵌入式底层驱动的工程化实践。项目全程遵循STM32标准开发流程与嵌入式工程规范，代码可直接编译运行，无额外硬件依赖，适合作为STM32入门级工程案例，同时具备完整的版本管理与问题排查记录，可直接同步至GitHub仓库。
 
技术栈：STM32CubeMX、HAL库、C语言、Git/GitHub
核心功能：TIM2定时器中断配置、100MHz系统时钟稳定配置、非阻塞式LED周期闪烁（500ms亮/500ms灭）
适用场景：STM32基础外设学习、嵌入式中断机制实践、工程化开发流程训练

二、标准化实验步骤
 
阶段1：CubeMX工程初始化配置
 
1. 新建工程并选型：打开STM32CubeMX，通过MCU筛选器精准选择STM32F411RET6芯片，创建空白工程，命名符合嵌入式工程命名规范，避免特殊字符与中文。
2. GPIO引脚配置：在芯片引脚视图中定位PA5引脚，将其设置为GPIO推挽输出模式，保持默认输出电平、上下拉电阻配置，仅完成基础输出功能配置，不做冗余参数修改。
3. RCC时钟源配置：进入系统核心RCC配置界面，关闭外部高速时钟HSE与外部低速时钟LSE，采用芯片内部HSI 16MHz时钟作为时钟源，保证无外部晶振时工程可稳定运行。
 ![[030128b1-0eeb-41ea-9011-e30d892ba16b.png]]![[Pasted image 20260325110608.png|533]阶段2：系统时钟树精准配置
 
1. 进入时钟配置页面，按照STM32F411RET6芯片时钟规范，配置PLL倍频分频参数：PLL时钟源选择HSI，PLLM分频系数设为8，PLLN倍频系数设为100，PLLP分频系数设为2，通过公式计算得出系统时钟SYSCLK为100MHz，符合芯片主频运行范围。
2. 总线分频配置：AHB预分频器设为1分频，APB1预分频器设为2分频（遵守APB1总线最大50MHz时钟限制），APB2预分频器设为1分频，确保各总线时钟合规，无时钟警告报错。
3. 时钟验证：确认页面顶部系统时钟显示为100MHz，无红色警示标识，完成稳定时钟配置。
 ![[db2244f3-e12f-40ca-ba5b-acacc9f4f7f6.png]]
阶段3：TIM2定时器与中断配置
 
1. 定时器基础配置：左侧外设栏选择TIM2定时器，时钟源设置为内部时钟，进入参数配置界面，预分频值PSC设为9999，自动重装载值ARR设为99，通过时钟频率计算实现10ms单次中断触发。![[Pasted image 20260325110804.png]]
2. 中断使能配置：切换至NVIC中断配置页面，勾选TIM2全局中断并开启，配置合理中断优先级，保证定时器中断可正常触发且不与系统其他中断冲突。
3. 代码生成设置：选择MDK-ARM作为编译环境，配置工程存储路径，生成初始化代码，确保CubeMX自动生成的驱动文件完整无缺失。
 
阶段4：Keil工程代码实现
 
1. 全局变量定义：在main.c文件头部，定义定时器中断计数全局变量 uint32_t tim2_count = 0; ，用于记录中断触发次数，实现周期计时。
2. 中断启动配置：在main函数中，完成系统初始化、外设初始化后，while(1)循环之前，调用 HAL_TIM_Base_Start_IT(&htim2); 函数开启TIM2定时器中断，同时添加错误处理逻辑，保证中断启动异常时可触发错误处理机制。
3. 中断回调函数编写：在main.c文件末尾，实现定时器溢出中断回调函数 HAL_TIM_PeriodElapsedCallback ，判断中断源为TIM2后，累加计数变量，当计数达到50次（即500ms）时，翻转PA5引脚电平，重置计数变量，完成LED闪烁逻辑。
4. 主循环设置：while(1)主循环保持空循环，不添加任何阻塞式代码，所有控制逻辑由中断异步触发，体现非阻塞式开发的工程优势。
 
阶段5：编译下载与Git版本管理
 
1. 工程编译：在Keil MDK中编译工程，确保实现0错误、0警告，符合嵌入式工程编译规范。
2. 程序下载：连接ST-Link下载器与核心板，将编译完成的程序下载至芯片，完成硬件调试。

三、实验问题排查与解决
 
问题1：时钟配置红色警告，自动修复后主频恢复默认值
 
问题现象：手动配置PLL参数后，时钟页面出现红色警告，点击自动修复后系统时钟退回16MHz，无法达到目标主频。
原因分析：仅修改PLLN与PLLP参数，未合理配置PLLM分频系数，导致时钟计算值超出芯片允许范围，CubeMX自动触发安全保护机制，重置时钟配置。
解决方法：严格按照时钟计算公式重新配置PLL参数，固定PLLM=8、PLLN=100、PLLP=2，保证时钟链路计算合规，手动完成配置后无警告，无需依赖自动修复功能。
工程经验：嵌入式系统时钟配置需先做理论计算，再进行参数配置，杜绝盲目修改，保证系统时钟稳定性。
 
问题2：程序下载后LED无反应，无闪烁现象
 
问题现象：编译下载无报错，但板载LED始终处于常亮或常灭状态，无周期性闪烁。
原因分析：一是未在NVIC配置中开启TIM2全局中断，中断链路未打通；二是未调用定时器中断启动函数，定时器仅完成初始化但未开启工作模式。
解决方法：返回CubeMX检查TIM2全局中断使能状态，确保勾选开启；在main函数中添加定时器中断启动函数，完善中断使能全流程。
工程经验：STM32外设中断需完成外设初始化、中断使能、功能启动三步流程，缺一不可，需养成全流程检查的工程习惯。
 
问题3：LED闪烁周期不准确，时序偏差较大
 
问题现象：LED闪烁频率与预期不符，周期过长或过短，时序控制不精准。
原因分析：系统时钟未实际生效，程序运行时仍采用默认16MHz时钟，导致定时器定时计算逻辑出错。
解决方法：通过Keil在线调试功能，查看系统时钟变量值，确认时钟配置生效；重新核对定时器PSC与ARR参数，确保与实际系统时钟匹配。
工程经验：时序相关功能需验证系统时钟实际运行值，参数配置需与硬件时钟匹配，保证时序精准性。
